c++ learning notes 2

1.关于c++函数
函数原型即函数声明
c++中函数原型参数为空即不接受参数eg.void fun(),而c中则是指不指定参数，若c++函数不指定参数应使用fun(…)
声明函数指针eg.int (p)(int)，定义函数指针为p=fun;//注意没有括号，不然就是调用fun了；
调用函数指针有两种方式，一种为(p)();另一种直接为p();//最好使用前一种，明显使用函数指针
内联函数声明声明和定义都要使用inline关键字，但通常可省略声明，eg.inline int fun();内联函数是用函数代码替换函数调用，因此更快，但是也更加占用内存空间；在c中虽然可以使用函数宏定义来实现类似功能，但没用inline好用和强大
函数可以有默认参数，通过函数声明实现，eg.int fun(int a,int b=1);//使得b的默认值为1，注意默认参数必须从右向左依次赋值，不能跳过也不能颠倒顺序，调用时必须从左至右，eg fun(1)相当于fun(1,1)

2.关于const
const可以用来声明常量eg.const a=100;
const可以用来限定不允许修改eg.const char p;那么不允许通过指针p来修改其指向的值
不能既定义const常量，有用非const的指针指向，eg.const int a=10;int p=&a;//非法
通过字符串常量定义的指针不能修改其字符串，因为存储在常量区，修改会引发程序崩溃
非const的变量可以赋值给const变量，反之不可以

3.关于c++中的引用
可以创建引用类型eg.int a;int & b=a;必须在声明时即赋值，而不能先声明后赋值
引用一但被创建，则始终与变量绑定在一起，相当于常量指针，但是更加高级的常量指针
可以通过指针给引用赋值，eg.int p=&a;int &b=p;此后b始终与a绑定
可以在函数中声明按引用传递，即传地址方式eg.int fun(int &a,int &b)，如果在参数为const &，那么将在必要时生成临时变量
引用可以被用于给结构体赋值，eg strcut_name & a=struct_var1;struct_var2=a;//这样可以避免创建新的拷贝，节省内存
const &的含义是不能通过该引用修改与其绑定的值
基类的引用可以指向派生类，即基类引用可以被赋值为派生类，eg ostream &a=ofstream b；//ofstream类是ostream类的派生类
4.关于c++中函数重载
重载与多态指的是一回事
重载通过函数参数的不同以实现同名函数的不同动作，而返回类型不同的同名函数是非法的，注意c++不将引用作为参数区别，即int &a和int a视作同种类型参数
注意const只在指针和引用时作为不同的参数，而不能区别一个const int和int

5.关于函数模板
template //class或typename关键字均可，只是typename是新加的
int fun(通用类型）
{…}//代码中涉及要替换的类型都要使用设置的通用类型名称
声明时为int fun(通用类型）；
函数模板也可用重载，关键在于是的函数参数个数不同eg.template int fun(Any,int);//多了一个int
函数模板的显示具体化，当用于特定情况时，可以使用函数模板具体化eg.
template <> int fun<特定类型>（特定类型）；//这是现代版本，如果是早期版本的显式具体化，则要使用int fun<>(特定类型）
显示实例化用于通过模板创建特定类型的函数,eg.template int fun<特定类型>(特定类型);//注意显示实例化声明比显示具体化少了一个<>，同时显示实例化由于实际创建了一个函数，加快了编译速度
在函数调用时，优先级别为非模板版本>显示实例化模板版本>显示具体化模板版本>模板版本

6.关于头文件
头文件里面通常放的是各种声明
在安装库中找eg#include，在程序文件夹找eg.include “head.h”

7.关于static
可以在代码块内部声明static变量，作用域在代码块
可以在代码块外部声明static变量(注意是整个代码块外部，包括main函数），作用域在整个文件（但外部文件无法引用）
可以在代码块外部声明变量，那么在本文件和其他文件中均可使用,外部文件使用extern关键字声明

8.关于::操作符
用于变量之前eg.::a，表明使用a的全局版本
用于名字地址空间eg:std::cout

9.关于多文件编译
可以将多个文件一起编译，此时需要在在调用函数的文件中写上函数声明
也可以include将要包含的文件，此时调用不需要写上函数声明
还有一个问题，c++的标准步骤是将函数声明放在头文件中，在主调文件包含头文件，并且主调文件应当与函数定义文件一起编译才行，那么为什么在使用诸如printf只要包含stdio.h呢，这是因为编译程序集成了标准库，在调用时编译器自动去找dll（windows下，在linux中去找so），就是动态链接了

10.关于volatile和mutable关键字
这两个用的比较少，volatile关键字告知编译器不要使用空间局部性优化，即不要将内存中频繁使用的变量放到cache
mutable关键字告知编译器即是某一结构体或其他东西声明为const，但其中的某些成员依然可以被更改
eg.strcuct data {char a[]=”hello”;mutable int b};const data st;//此时可以修改st.b但不能修改st.char[]

11.关于extern
一般const定义的全局变量不能被外部引用，eg.const int a;但此时若加上extern则可以使a能被外部引用
要引用外部文件定义的变量时要使用extern关键字,eg.extern a;//告知编译器a被定义在另一个文件

12.关于new和布局new
new用来在堆中分配内存，必须要与delete成对使用，但要注意delete并没有抹掉内存值，而是告诉编译器可以回收这块内存
布局new用于在指定内存地址开辟空间，首先要包含头文件，使用方法eg.new (要开始放的地址）类型

13.关于名称空间namespace
名称空间必须定义在代码块外部，也可以位于另一个名称空间中，是开放的，eg.namespace std{string test=”namespace”;}就是将test加入到std中
引用名称空间用::或者using，using可以使得一个名称可用eg.using std::cout//以下就可以用cout代替std::cout;也可以using namespace std;//这将引用std中的所有名称
可以给名称空间起别名，eg.namespace name=std;
